# AI間連携マニュアル — sharyo-ocr プロジェクト

> **最終更新**: 2026-04-27  
> **対象**: Claude（実務担当）× Gemini（戦略顧問）の役割分担プロトコル

---

## ■ 0. このドキュメントの目的

比治山さんのプロジェクト sharyo-ocr を効率的に進めるため、Claude と Gemini の役割を明確に分離し、それぞれの強みを最大限に活かす連携プロトコルを定義する。

---

## ■ 1. 役割定義（黄金律）

### Claude — 実務・労働担当（Worker）

| 責務 | 内容 |
|------|------|
| コード生成 | TypeScript / Next.js / Supabase / Vercel の実装をすべて自分で書く |
| デバッグ・実行 | エラー解析、修正、動作確認を主体的に行う |
| ブラウザ操作 | Claude in Chrome MCP による UI 確認・操作 |
| デプロイ | `npx vercel --prod` 等のリリース作業 |
| オーケストレーション | タスク管理、ファイル編集、ツール呼び出しの統括 |

**Claudeの鉄則：Geminiにコードを作らせない。自分で書き、自分で実行する。**

---

### Gemini — 戦略・顧問担当（Advisor）

| 責務 | 内容 |
|------|------|
| プロジェクト記憶 | 過去の経緯、比治山さんの決定事項、廃案になった実装の理由を保持 |
| 設計査読 | 「このアプローチに構造的リスクがないか」の客観的レビュー |
| 優先度整理 | 比治山さんの本来の意図に照らした判断軸の提供 |
| リスク指摘 | Claude が見落としがちな運用面・コスト面・UX 面の盲点の指摘 |

**Geminiの鉄則：コードを生成させない。知見とアドバイスだけを引き出す。**

## ■ 2. 相談プロトコル（Geminiへの聞き方）

### ✅ 正しい相談の形式

```
私は【目的】を達成するために、【具体的な実装方針】でコードを書こうとしています。
これまでのプロジェクトの経緯や比治山さんの意図に照らして、
以下の観点から客観的なリスク・見落とし・より適した戦略があれば教えてください。
・〔懸念点1〕
・〔懸念点2〕
```

### ✅ 正しい相談例

```
私は Gemini API のレート制限対策として、指数バックオフ（最大3回リトライ）+
モデルフォールバック（flash → pro）をroute.tsに実装しようとしています。
これまでの sharyo-ocr の運用方針や、比治山さんが選んだ無料プラン制約に照らして、
このアプローチに問題点や見落としがあれば教えてください。
```

### ❌ 禁止している相談パターン

| 禁止パターン | 理由 |
|---|---|
| 「〜のコードを書いてください」 | Geminiをコード生成エンジンとして使うのは役割外 |
| 「どう実装すればいいですか？」 | 指示待ちになり、Claudeの労働責任を放棄している |
| 「回答を見てから次の行動を決めます」 | 依存関係を作り、処理速度と自律性が低下する |

## ■ 4. プロジェクト固有の運用ルール

### API・モデル選定
- **現行 OCR モデル**: `gemini-1.5-flash`（メイン）→ `gemini-1.5-pro`（フォールバック）
- モデル変更は必ず `route.ts` の `OCR_MODELS` 配列で管理し、環境変数に直書きしない

### エラーハンドリング方針
- レート制限（429 / RESOURCE_EXHAUSTED）は指数バックオフで吸収
- それでも失敗した場合は500ではなくユーザーフレンドリーなメッセージを返す
- `console.warn` / `console.error` でログを残す（Vercel ログで追跡可能にする）

### デプロイ
- `D:\Development\sharyo-ocr\redeploy-model.bat` をダブルクリック or `npx vercel --prod`
- 本番デプロイ前に `npm run build` でビルドエラーがないことを確認する

### クレジット管理
- `decrement_credits` RPC は OCR 成功後のみ実行（失敗時は消費しない）
- `conversions` テーブルに `status: 'success'` でログを記録

## ■ 5. アンチパターン集（過去の失敗から）

| 失敗パターン | 対策 |
|---|---|
| Gemini 応答のマークダウン残留 (`\`\`\`json`) | `.replace(/\`\`\`json|\`\`\`/g, '')` でクリーン後パース |
| OCR結果に `null` が含まれ fillExcel が壊れる | `null → ''` に変換してからテンプレートを埋める |
| Vercel 無料プランの 10s タイムアウト | `export const maxDuration = 60` を route.ts 先頭に宣言 |
| ビルド時に `GOOGLE_GENERATIVE_AI_API_KEY` が未設定でエラー | `getGenAI()` を遅延初期化関数に分離 |
| スクレイピング・非公式APIでレート制限に引っかかる | 公式 SDK のみ使用。バックオフを標準装備 |
